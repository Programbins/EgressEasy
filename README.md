# EgressEasy: 초기 행동 연계형 비상구 인지 시스템

## 1. 프로젝트 배경

2020년부터 2024년까지 다중이용업소에서 발생한 화재는 총 1,948건으로, 10명의 사망자와 172명의 부상자를 낳았습니다. 이는 건수 대비 인명 피해 비율 9.3%에 달하는 수치로, 일반 건축물(6.2%) 대비 1.5배 높은 심각한 수준입니다.

화재 원인 분석 결과, 부주의가 40.7%(792건)로 가장 높았으나, **실제 이용객의 비상구 인지율이 10% 미만**이라는 점이 인명 피해를 키우는 결정적 요인으로 작용했습니다. 비상구 위치를 몰라 대피 '골든타임'을 놓치는 것이 가장 큰 문제였습니다.

본 프로젝트는 이러한 문제의식에서 출발했습니다. 다중이용업소 방문 시 **와이파이 접속, PC 로그인 등 사용자의 필수적인 초기 행동**을 비상구 인지 과정과 자연스럽게 연계하여, 화재와 같은 위급 상황 발생 시 인명 피해를 최소화하는 것을 목표로 합니다.

## 2. 시스템 목표

**"시설 진입 첫 순간, 가장 자연스러운 행동으로 비상 대피 경로를 체득하게 한다."**

본 시스템은 사용자가 와이파이에 접속하거나 좌석의 키오스크를 이용하는 바로 그 순간, 짧은 경로 안내 경험을 통해 자신의 현재 위치와 가장 가까운 비상구까지의 경로를 무의식적으로 인지하도록 설계되었습니다. 이를 통해 화재 발생 시 대피 초기 지연을 결정적으로 줄이고자 합니다.

## 3. 핵심 기능 및 구현 내용

- **위치 자동 판별 (SSID/VLAN Zone Mapping)**
  - 시설 내에 분산 설치된 네트워크 장비(AP)의 SSID와 VLAN을 각 비상구 구역(Zone)과 매핑합니다.
  - 사용자가 특정 구역의 와이파이에 접속하는 순간, 시스템은 별도의 절차 없이 사용자의 현재 위치(비상구)를 자동으로 식별합니다.

- **직관적인 경로 체험 제공**
  - 위치가 식별되면, 해당 비상구에서부터 사용자의 목적지(예: 좌석)까지의 최단 대피 경로가 담긴 맞춤형 안내(영상 또는 인터랙티브 맵)가 재생됩니다.
  - 사용자는 이를 통해 '내 자리에서 가장 가까운 비상구는 여기구나'를 자연스럽게 학습합니다.

- **안전 피로도 감소를 위한 지능형 DB 연동**
  - 중앙 데이터베이스에서 사용자의 계정 및 기기 정보를 기반으로 시설 방문 및 안내 시청 이력을 관리합니다.
  - 재방문 사용자의 경우, 불필요한 반복 안내를 건너뛰어(Skip) 사용자의 피로감을 낮추고 서비스 만족도를 높입니다.

- **저비용 및 확장성**
  - 고가의 신규 장비 도입 없이, 기존에 사용하던 공유기 등 네트워크 장비의 **멀티 SSID, VLAN, 캡티브 포털(Captive Portal)** 기능을 그대로 활용합니다.
  - 표준화된 템플릿을 통해 새로운 시설에도 빠르고 유연하게 시스템을 확장 적용할 수 있습니다.

## 4. 시스템 구성 및 흐름

**사용자 접속 → 구역 감지 → 맞춤 안내 → 임시 인증 → DB 연동**

1.  **멀티 SSID/VLAN/Zone 매핑**: 관리자가 시설 내 AP의 SSID를 각 비상구 구역과 매핑하여 설정합니다.
2.  **구역 감지 및 맞춤 영상 재생**: 사용자가 특정 SSID에 접속하면, 해당 구역에 맞는 비상구 안내 콘텐츠가 캡티브 포털을 통해 제공됩니다.
3.  **임시 인증**: 사용자는 안내를 확인한 후, 임시 인증을 통해 인터넷에 접속하거나 키오스크를 활성화합니다.
4.  **중앙 DB 연동**: 사용자의 기기/계정 정보와 안내 확인 이력이 중앙 DB에 기록됩니다.
5.  **서비스 확장**: 이 DB는 PC 클라이언트, 모바일 앱, 웹 등 다양한 플랫폼과 공유되어 일관된 사용자 경험을 제공합니다.
